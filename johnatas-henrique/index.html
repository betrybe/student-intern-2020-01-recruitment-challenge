Event Listeners

No início era...

Bom, pra começarmos, que tal entender a estrutura básica de um site e depois o que é um Event (evento), sabendo isso, o resto ficará muito mais fácil. Normalmente para fazer um site, temos 3 itens:

  • 1) HTML, que normalmente é o responsável pelo conteúdo em si do site e a estruturação do mesmo.
  • 2) CSS, que responde pela estilização (tamanhos, cores, posicionamento dos itens...) do site.
  • 3) JavaScript, que faz a magia negra criação de conteúdo que se atualiza dinamicamente, dá vida a caixas de texto, botões, imagens e por aí vai, quase tudo o que parece "vivo" em uma página, é feito com JavaScript ou JavaScript está envolvido.

Após saber mais sobre a tríade que cria os sites, você também precisa saber que o JavaScript consegue trabalhar o CSS e o HTML dinamicamente, em tese, um programador poderia fazer todo um site apenas com JS (apesar de ser mais tranquilo para um iniciante usar um pouco de cada uma das três), não façam isso em casa.

Também é importante saber o conceito de DOM, (Document Object Model), porque é através dele que você consegue utilizar todo o poder que o JS pode te proporcionar, incluindo os Event Listeners que ainda não sabemos o que é, não é mesmo? O DOM é um modelo para criar, alterar, pegar e deletar os elementos na tela, então o DOM é a base que diz ao seu código JavaScript qual elemento que é pra ele causar influência.

Ok, mas porque preciso saber de tudo isso antes, se eu só queria saber o que é um Event Listener?

Todo evento, trabalha em cima do DOM, então se você não tiver idéia do que é o DOM, você simplesmente não tem como escolher que elemento quer alterar, ou então, onde quer inserir seu novo elemento.

Mas e o Event Listener?

Calma lá que estamos chegando, no caso, todo elemento que você coloca na página (um bloco (div), um parágrafo de texto (p), uma caixa de texto (input), pode ser acessado pelo seu próprio nome de elemento, por uma classe CSS ou por um ID CSS, então a maneira que você escreve seu HTML, pode te ajudar muito a trabalhar mais facilmente com o JS.
Por exemplo, imagine que você tem um formulário com 10 campos (Nome, Sobrenome, Endereço, CEP...) e você queira fazer alguma interação apenas com o campo do nome? Concorda que ter um jeito fácil de escolher apenas o campo input do nome iria ajudar muito na tarefa? Então lhe apresento o comando: document.querySelector() Vamos lá, o que isso faz:
Esse comando é o pegador de objetos do JavaScript, lembra que falamos sobre pegar um elemento por id ou classe, dentro dos parenteses desse comando, vc digita o ID ou classe e ele te retorna o elemento para utilizar, temos abaixo o exemplo de uma caixa input e do código dela em HTML, onde podemos ver a ID da caixa e onde colocamos o ID no comando, um bônus seria colocar o resultado dentro de uma variável, como no exemplo abaixo: HTML:
' <' type="text" value="Trybe" id="caixa-de-texto">

JavaScript:
const caixaTrybe = document.querySelector('#caixa-de-texto');

Agora sim, é pra valer, partimos para mais uma Corrida Maluca vamos falar de Eventos!

O que é um evento?

Um jeito de entender os eventos em JavaScript para uma página web, seria relacionar os eventos aos nervos do corpo humano, onde todo estímulo pode (ou não), gerar uma reação.

Cada coisa que você faz dentro de uma página web (apertar alguma tecla no teclado, clicar num item, preencher uma caixa...), é um evento, e agora a explicação sobre Event Listener:

Event Listener é um código colocado em um elemento que fica ali esperando pra ver algum evento acontecer, um exemplo do mundo real, seria alguém que fica escondido, na surdina, esperando pra ver se alguma coisa acontece.

A primeira vez...

Depois de toda essa conversa (e luta), já estamos tão mais fortes que o monstro é... bem fraquinho

O comando para criar um escutador de eventos em um item, é addEventListener(), como toda função, ele recebe parâmetros, que são dois:

  • 1) O evento que estamos esperando escutar: Exemplos: click, change, mouseover...
  • 2) A função que será executada quando o evento acontecer.

Utilizando o elemento do exemplo acima, a escrita do código JavaScript é a seguinte: caixaTrybe.addEventListener('nome do evento', função que será chamada)

Para você começar, clique abaixo na imagem (que é um evento), e ganhe uma caixa com 2 eventos que você pode utilizar, além de um link externo para mais elementos, assim você poderá começar a utilizar Event Listeners:

Exemplos de eventos
click = Os cliques dados no mouse são capturados e executam a função (como o que você acabou de fazer).
change = Ao alterar o valor de um elemento (exemplo, uma caixa input), a função passada é executada.

Entendendo a mágica

O código do event listener feito acima está aqui abaixo, vamos passar por ele comentando rapidamente cada linha: const imgTakeThis = document.querySelector('#take-this');
const divTakeThis = document.querySelector('#box-take-this');
function abreCaixa () {
divTakeThis.classList.remove('esconder');
}
imgTakeThis.addEventListener('click', abreCaixa)
 A primeira e a segunda linha, pegamos os elementos que queremos trabalhar, no caso, a imagem que clicamos (imgTakeThis) e a div que queremos que apareça (divTakeThis).
Perceba, que você não é obrigado a fazer alterações no mesmo elemento onde capturou o evento, no nosso caso, capturamos o evento de clique na imagem e alteramos uma div que não tem ligação direta com a imagem.
Da terceira a quinta linha, temos a função que está sendo executada quando o evento ocorre, essa função está removendo uma classe, (trabalhando com CSS dinamicamente) que era a responsável por esconder a div na página.
E na sexta linha, temos o event listener, sendo colocado na imagem, passando o parametro click, e a função que queremos executar quando o evento acontece.
