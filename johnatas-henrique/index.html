Event Listeners

Coisas importantes a saber pra começar

Bom, pra começarmos, que tal entender a estrutura básica de um site e depois o que é um Event (evento), sabendo isso, o resto ficará muito mais fácil. Normalmente para fazer um site, temos 3 itens:

  • 1) HTML, que normalmente é o responsável pelo conteúdo em si do site e a estruturação do mesmo.
  • 2) CSS, que responde pela estilização (tamanhos, cores, posicionamento dos itens...) do site.
  • 3) JavaScript, que faz a magia negra criação de conteúdo que se atualiza dinamicamente, dá vida a caixas de texto, botões, imagens e por aí vai, quase tudo o que parece "vivo" em uma página, é feito com JavaScript ou JavaScript está envolvido.

Após saber mais sobre a tríade que cria os sites, você também precisa saber que o JavaScript consegue trabalhar o CSS e o HTML dinamicamente, em tese, um programador poderia fazer todo um site apenas com JS (apesar de ser mais tranquilo para um iniciante usar um pouco de cada uma das três), não façam isso em casa. Também é importante saber o conceito de DOM, (Document Object Model), porque é através dele que você consegue utilizar todo o poder que o JS pode te proporcionar, incluindo os Event Listeners que ainda não sabemos o que é, não é mesmo? O DOM é um modelo para criar, alterar, pegar e deletar os elementos na tela, então o DOM é a base que diz ao seu código JavaScript qual elemento que é pra ele causar influência.

Ok, mas porque preciso saber de tudo isso antes, se eu só queria saber o que é um Event Listener?
A resposta, é que o Event, trabalha em cima do DOM, então se você não tiver idéia do que é o DOM, você simplesmente não tem como escolher que elemento quer alterar, ou então, onde quer inserir seu novo elemento (depois de qual elemento existente quer colocar o seu novo, ou dentro de qual elemento quer colocar o seu elemento novo).

Ok, mas e o Event Listener?

Calma lá que estamos chegando, no caso, todo elemento que você coloca na página (um bloco (div), um parágrafo de texto (p), uma caixa de texto (input), pode ser acessado pelo seu próprio nome de elemento, por uma classe CSS ou por um ID CSS, então a maneira que você escreve seu HTML, pode te ajudar muito a trabalhar mais facilmente com o JS. Por exemplo, imagine que você tem um formulário com 10 campos (Nome, Sobrenome, Endereço, CEP...) e você queira fazer alguma interação apenas com o campo do nome? Concorda que ter um jeito fácil de escolher apenas o campo input do nome iria ajudar muito na tarefa? Então lhe apresento o comando: document.querySelector() Vamos lá, o que isso faz:
Esse comando é o pegador de objetos do JavaScript, lembra que falamos sobre pegar um elemento por id ou classe, dentro dos parenteses desse comando, vc digita o ID ou classe e ele te retorna o elemento para utilizar, temos abaixo o exemplo de uma caixa input e do código dela em HTML, onde podemos ver a ID da caixa e onde colocamos o ID no comando, um bônus seria colocar o resultado dentro de uma variável, como no exemplo abaixo: HTML:
'<'type="text" value="Trybe" id="caixa-de-texto">

JavaScript:
const caixaTrybe = document.querySelector('#caixa-de-texto');

Agora sim, com esse background todo, partiu Eventos!
